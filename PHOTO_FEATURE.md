# Функція фото товарів

## Опис

Дана функція дозволяє користувачам завантажувати до 3 фотографій для кожного товару. Фото відображуються:
- На картці товару (стікер з кількістю)
- У модальному вікні додавання товару (фотогалерея)

**Важливо:** Фото використовуються тільки для візуалізації при додаванні товарів у список. Вони **не включаються** в звіти, статистику або експортовані файли.

## Основні можливості

### Для користувачів

1. **Завантаження фото**
   - Максимум 3 фото на товар
   - Автоматичне стискання до ~500KB
   - Підтримка JPEG, PNG та інших форматів

2. **Перегляд фото**
   - Гортання між фото стрілками
   - Повноекранний режим (клік по фото)
   - Лічильник фото (1/3, 2/3, тощо)

3. **Візуальні індикатори**
   - Золотий стікер на картці товару (📷 2)
   - Відображення кількості завантажених фото

### Для адміністраторів

1. **Модерація фото**
   - Черга очікуючих фото (pending)
   - Швидке схвалення/відхилення
   - Вказання причини відхилення

2. **Керування фото**
   - Видалення недопустимих фото
   - Перегляд статистики (завантажено/схвалено/відхилено)
   - Відстеження авторів завантажень

## Технічні деталі

### Структура бази даних

```sql
CREATE TABLE product_photos (
    id SERIAL PRIMARY KEY,
    артикул VARCHAR(20) REFERENCES products(артикул),
    file_path VARCHAR(500) NOT NULL,
    file_size INTEGER NOT NULL,
    original_size INTEGER NOT NULL,
    photo_order INTEGER DEFAULT 0,
    uploaded_by BIGINT REFERENCES users(id),
    uploaded_at TIMESTAMP DEFAULT NOW(),
    status VARCHAR(20) DEFAULT 'pending',
    moderated_by BIGINT REFERENCES users(id),
    moderated_at TIMESTAMP,
    rejection_reason VARCHAR(500)
);
```

### Стискання зображень

- **Максимальний розмір:** 1200px по більшій стороні
- **Цільовий розмір файлу:** ~500KB
- **Формат:** JPEG з оптимізацією
- **Метод:** Прогресивне зменшення якості (95% → 50%)

### API ендпоінти

#### Завантаження фото
```
POST /api/photos/upload
Content-Type: multipart/form-data

Body:
- photo: File
- article: string
- user_id: integer
```

#### Отримання фото товару
```
GET /api/photos/product/{article}

Response: {
  success: true,
  photos: [
    {id: 1, file_path: "...", order: 0},
    ...
  ]
}
```

#### Модерація
```
GET /api/photos/moderation/pending?user_id={admin_id}
POST /api/photos/moderation/{photo_id}
  - status: "approved" | "rejected"
  - reason: string (optional)
  - user_id: integer
```

## Використання

### Для користувачів

1. Відкрити модальне вікно додавання товару
2. Натиснути "📷 Додати фото"
3. Обрати фото з галереї або зробити нове
4. Фото автоматично стиснеться і надіслеться на модерацію

### Для адміністраторів

1. Відкрити вкладку "🛠️ Адмін"
2. Прокрутити до секції "🖼️ Модерація фото"
3. Переглянути фото та натиснути:
   - "✅ Схвалити" — фото стане видимим для всіх
   - "❌ Відхилити" — вказати причину

## Обмеження

- Максимум 3 фото на один товар
- Максимальний розмір завантажуваного файлу: 10MB
- Підтримувані формати: JPEG, PNG, WebP, GIF
- Фото використовуються **тільки** для перегляду при додаванні

## Безпека

- Всі фото проходять модерацію
- Доступ до функцій модерації тільки для адміністраторів
- Зберігається історія завантажень (хто, коли, який статус)
