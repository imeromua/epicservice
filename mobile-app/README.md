## 📝 Опис

EpicService — це автономний Android додаток, створений на основі діючого Telegram мініаппу. Додаток працює повністю офлайн, використовуючи локальну SQLite базу даних. Авторизація за логіном та паролем з підтримкою біометрії.

### Ключові відмінності від мініаппу:
- ❌ Без Telegram API та Telegram авторизації
- ✅ Локальна авторизація (логін/пароль)
- ✅ Біометрична автентифікація (відбиток пальця / Face ID)
- ✅ Повністю офлайн — власна SQLite база даних
- ✅ Незалежний від сервера чи домену
- ✅ Нативний Android додаток (React Native + Expo)

## ⭐ Функціонал

### Авторизація
- 🔐 Вхід за логіном та паролем
- 📝 Реєстрація нових користувачів
- 🔒 Перший зареєстрований користувач автоматично стає адміністратором
- 👆 Біометрична автентифікація (відбиток пальця)
- 🔑 Безпечне збереження сесії (expo-secure-store)

### Пошук товарів (🔍)
- Пошук за назвою або артикулом
- Автоматичний пошук з затримкою 300мс (debounce)
- Фільтрація за відділами
- Відображення ціни та деталей товару
- Додавання до списку з вибором кількості

### Управління списком (📋)
- Перегляд поточного списку
- Зміна кількості товарів (+/-)
- Видалення окремих товарів
- Групування за відділами
- Підрахунок загальної суми
- Збереження списку до архіву
- Очищення всього списку

### Архіви (📁)
- Перегляд збережених списків
- Розгортання/згортання деталей архіву
- Дата та кількість позицій
- Видалення архівних записів

### Адмін панель (⚙️)
- 📊 Статистика (кількість товарів та користувачів)
- 📥 Імпорт товарів з CSV файлу
- 🔐 Налаштування біометричного входу
- ⚠️ Очищення бази товарів
- 🔴 Повне очищення даних (з підтвердженням «ВИДАЛИТИ ВСЕ»)

## 🚀 Встановлення та запуск

### Передумови

- **Node.js** 18+ (рекомендовано 20+)
- **npm** 9+
- **Expo CLI** (встановлюється автоматично через npx)
- **Android Studio** (для емулятора або збірки APK)
- **JDK 17** (для збірки Android)

### Крок 1: Встановлення залежностей

```bash
cd mobile-app
npm install
```

### Крок 2: Запуск у режимі розробки

```bash
# Запуск Expo dev server
npx expo start

# Або для Android безпосередньо
npx expo start --android
```

### Крок 3: Запуск на емуляторі

1. Відкрийте Android Studio
2. Запустіть емулятор (AVD Manager)
3. Виконайте `npx expo start --android`

### Крок 4: Запуск на пристрої

1. Встановіть **Expo Go** з Google Play
2. Виконайте `npx expo start`
3. Відскануйте QR-код з терміналу

## 📦 Збірка APK

### Локальна збірка (без Expo Cloud)

#### Передумови для локальної збірки:

1. **Android Studio** з Android SDK
2. **JDK 17**
3. Змінні середовища:
   ```bash
   export ANDROID_HOME=$HOME/Android/Sdk
   export PATH=$PATH:$ANDROID_HOME/emulator
   export PATH=$PATH:$ANDROID_HOME/platform-tools
   ```

#### Крок 1: Генерація нативного проєкту

```bash
cd mobile-app
npx expo prebuild --platform android
```

Це створить директорію `android/` з нативним Android проєктом.

#### Крок 2: Збірка APK (debug)

```bash
cd android
./gradlew assembleDebug
```

APK файл буде створено у:
```
android/app/build/outputs/apk/debug/app-debug.apk
```

#### Крок 3: Збірка APK (release)

```bash
# Створення ключа підпису (одноразово)
keytool -genkey -v -keystore epicservice.keystore \
  -alias epicservice -keyalg RSA -keysize 2048 -validity 10000 \
  -storepass ваш_пароль -keypass ваш_пароль \
  -dname "CN=EpicService, OU=Dev, O=EpicService, L=Kyiv, S=Kyiv, C=UA"

# Збірка release APK
cd android
./gradlew assembleRelease
```

APK файл буде створено у:
```
android/app/build/outputs/apk/release/app-release.apk
```

### Збірка через EAS (Expo Application Services)

```bash
# Встановлення EAS CLI
npm install -g eas-cli

# Авторизація
eas login

# Збірка APK
eas build --platform android --profile preview

# Або збірка AAB для Google Play
eas build --platform android --profile production
```

#### Конфігурація EAS (eas.json)

Створіть файл `eas.json` у корені `mobile-app/`:

```json
{
  "cli": {
    "version": ">= 5.0.0"
  },
  "build": {
    "development": {
      "developmentClient": true,
      "distribution": "internal"
    },
    "preview": {
      "android": {
        "buildType": "apk"
      },
      "distribution": "internal"
    },
    "production": {
      "android": {
        "buildType": "app-bundle"
      }
    }
  }
}
```

## 📄 Формат CSV для імпорту

### Структура файлу

```csv
артикул,назва,відділ,група,кількість,ціна
АРТ001,Молоток ударний,Інструменти,Ручні інструменти,50,185.00
АРТ002,Дриль акумуляторна,Електроінструменти,Дрилі,25,3450.00
АРТ003,Викрутка хрестова,Інструменти,Ручні інструменти,100,45.50
```

### Правила:

- **Роздільник**: кома (`,`) або крапка з комою (`;`)
- **Перший рядок**: може бути заголовком (автоматично визначається)
- **Мінімум полів**: артикул та назва (обов'язкові)
- **Ціна**: крапка або кома як роздільник десятих
- **Кодування**: UTF-8
- **Поля в лапках**: підтримуються подвійні лапки (`"`)

## 🔐 Авторизація та безпека

### Зберігання паролів
- Паролі хешуються алгоритмом **SHA-256** через `expo-crypto`
- Оригінальний пароль ніколи не зберігається

### Сесії
- Зберігаються у **expo-secure-store** (шифроване сховище Android Keystore)
- Автоматичний вхід при повторному відкритті додатку

### Біометрія
- Використовується **expo-local-authentication**
- Підтримка відбитка пальця та Face ID
- Вмикається/вимикається через адмін панель
- Біометричні дані залишаються на пристрої

### Ролі
| Роль | Доступ |
|---|---|
| `admin` | Всі функції + адмін панель |
| `user` | Пошук, список, архіви |

## 🗄️ Архітектура бази даних

### SQLite таблиці

```
┌──────────────┐     ┌──────────────────┐
│    users     │     │    products      │
├──────────────┤     ├──────────────────┤
│ id (PK)      │     │ id (PK)          │
│ login        │     │ article          │
│ password_hash│     │ name             │
│ first_name   │     │ department       │
│ role         │     │ group_name       │
│ status       │     │ quantity         │
│ created_at   │     │ price            │
└──────┬───────┘     │ active           │
       │             └────────┬─────────┘
       │                      │
       ▼                      ▼
┌──────────────┐     ┌──────────────────┐
│  temp_list   │     │  saved_lists     │
├──────────────┤     ├──────────────────┤
│ id (PK)      │     │ id (PK)          │
│ user_id (FK) │     │ user_id (FK)     │
│ product_id   │     │ name             │
│ quantity     │     │ created_at       │
└──────────────┘     └────────┬─────────┘
                              │
                              ▼
                     ┌──────────────────┐
                     │saved_list_items  │
                     ├──────────────────┤
                     │ id (PK)          │
                     │ list_id (FK)     │
                     │ article          │
                     │ product_name     │
                     │ quantity         │
                     └──────────────────┘
```

### Індекси
- `idx_products_name` — пошук за назвою
- `idx_products_article` — пошук за артикулом
- `idx_products_department` — фільтр за відділом
- `idx_temp_list_user` — список користувача
- `idx_saved_lists_user` — архіви користувача
